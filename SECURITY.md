# Security Policy

## Supported Versions

CrashHive32 security reports should target the latest published release or the current main branch.

## Reporting a Vulnerability

Please report security concerns privately to: parvaz.nic@gmail.com

Do not open a public issue for sensitive vulnerabilities.

## Scope Notes

CrashHive32 is not a safety-certified industrial protection system. It is a developer library for observability, diagnostics, and fault awareness.

The library does not currently implement ESP-NOW encryption, key management, MQTT, backend services, dashboards, or AI/API key handling.
